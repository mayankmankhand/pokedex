Design Notes

Guardrail applied: "Chrome bold, data restrained." In every view, the theme lives in borders, headers, badges, and accent elements. Data cells, descriptions, and field values stay in neutral colors with no decorative treatment.
Detail View decision: You have 3 options. A is the lightest touch (upgrade of current). B adds Pokedex-style framed cards. C uses a colored accent bar and horizontal stat layout. Pick whichever feels right - they all follow the same guardrail.
Audit log tone: Bold uppercase actor + action verb, but no battle narration. "BROCK APPROVED TP-003" not "BROCK used APPROVE! It's super effective!" Professional with personality.
New elements: Source badges on audit entries (chat/panel/api), Pokedex # numbering on tables, Gen 1 arrow-cursor confirmation box, fainted Pokeball error state. All CSS-only, no external assets.
What stays the same: Hybrid palette, status badge colors, panel frame structure, fonts (DM Sans + JetBrains Mono), keyboard shortcuts, drag-to-resize, navigation history. Phase 2 themes the content inside the panel, not the panel shell.
Accessibility: All new color combos checked for WCAG AA contrast. No new animations added (existing Pokeball spinner already respects prefers-reduced-motion). Arrow cursor in confirmation box is text-based, not image-based.
