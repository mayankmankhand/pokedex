Design Notes

Fonts: DM Sans + JetBrains Mono stay as-is. No pixel fonts - readability matters in a data-heavy PLM app.
Status Badges: Mapped to Pokemon type colors (Draft=Normal, Approved=Grass, Active=Water, Failed=Fire, Blocked=Poison, Canceled=Ghost, Skipped=Rock, Pending=Ground, Passed=Grass). Text labels are the primary communicator - color is an accent layer, not the only signal.
Pokeball Button: Pure CSS - no images. The send button uses a linear gradient (red top, band, white bottom) with a center circle. Hovering scales it slightly. Subtle but recognizable.
Loading: One Pokeball spinner (CSS rotation). Thinking indicator cycles Pokemon battle phrases. prefers-reduced-motion disables the spin and shows a static Pokeball + text only.
Guardrails: "Chrome bold, data restrained." Tables and detail fields stay calm. The theme lives in headers, borders, buttons, and badges - not in the data cells.
What's NOT in this prototype: Battle encounter transitions, trainer sprite avatars, favicon changes, typewriter effects, Game Boy keyboard hints, bespoke panel redesigns. Those are Phase 2 and 3.
